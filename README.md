1. Логические ошибки в реализации грамматики
Проблема: Неверная интерпретация грамматики
Что было:

cpp
int NumRead(string& s, int& symb, int currentNumber = 0) {
    if (symb >= s.length() || !isdigit(s[symb])) {
        return currentNumber;
    }
    currentNumber = currentNumber * 10 + (s[symb] - '0');
    symb++;
    return NumRead(s, symb, currentNumber);
}
Функция NumRead пыталась читать многозначные числа, но по грамматике <цифра> означает одну цифру (0-9), а не многозначное число. Выражения вида 12+3 недопустимы по условию, только 1*2+3.

Исправление: Убрана функция NumRead. Терм теперь читает одну цифру и при умножении рекурсивно умножает её на следующий терм.

Проблема: Добавление несуществующей операции
Что было:

cpp
if (znak == '*') {
    value *= number;
} else {
    if (number == 0) {
        cout << "Ошибка при делении на ноль!" << endl;
        exit(1);
    }
    value /= number;
}
В коде присутствовала обработка операции деления /, хотя в грамматике её нет. Также была проверка деления на ноль для несуществующей операции.

Исправление: Убрана обработка деления полностью.

2. Нарушение главного требования задачи
Проблема: Использование циклов
Что было:

cpp
int CalculateTerm(string& s, int& symb) {
    int value = NumRead(s, symb);
    while (symb < s.length() && (s[symb] == '*' || s[symb] == '/')) {
        // обработка
    }
    return value;
}

int CalculateS(string& s, int& symb) {
    int value = CalculateTerm(s, symb);
    while (symb < s.length() && (s[symb] == '+' || s[symb] == '-')) {
        // обработка
    }
    return value;
}
Обе функции использовали циклы while, что прямо запрещено условием задачи.

Исправление: Замена всех циклов на рекурсию:

Для термов: рекурсивный вызов CalculateTerm при обнаружении *

Для выражений: рекурсивный вызов CalculateExpression при обнаружении + или -

3. Стилистические ошибки (Google Code Style)
Проблема: Использование using namespace std;
Что было: using namespace std;

Исправление: Убрана директива, добавлены префиксы std:: ко всем элементам из стандартной библиотеки:

std::cout, std::cin, std::endl

std::string, std::getline

std::setlocale

Проблема: Неинформативные имена функций
Что было:

CalculateS - непонятно, что означает S

NumRead - слишком общее название

Исправление:

CalculateS → CalculateExpression (явно указывает, что вычисляет выражение)

Функция NumRead удалена как ненужная

Проблема: Отсутствие отступов
Что было: Несогласованные отступы

Исправление: Приведены отступы к 2 пробелам, открывающие скобки на той же строке

5. Ошибки в работе со строкой
Проблема: Неправильные имена переменных
Что было:

cpp
string S;  // заглавная буква
getline(cin, S);
Исправление: Переименовано в s (строчная буква) для соответствия Google Style и единообразия

Проблема: Отсутствие проверки границ строки
Что было: Недостаточно проверок выхода за пределы строки

Исправление: Добавлены проверки if (symb >= s.length()) в начале каждой рекурсивной функции
